> The frontend is in a separate repository: [Sequence-Companion-App-client](https://github.com/mikeislas06/Sequence-Companion-App-client).

## Technologies

- **Runtime:** Node.js 18+
- **Language:** TypeScript
- **Framework:** Express
- **Real-time:** Socket.IO v4
- **CORS:** cors
- **Dev tools:** Nodemon, ts-node
- **Testing:** Jest + ts-jest
- **Build:** TypeScript compiler (`tsc`)

## Features

- Room creation and join with short invite codes
- Auto team assignment across two teams
- Card deck management: shuffle, deal, and play
- Dead card replacement
- Sequence row tracking with win detection
- Disconnect handling: players removed and room state rebroadcast
- Game reset (play again with same room)
- `/health` endpoint for uptime checks

## Prerequisites

- Node.js 18+
- npm or pnpm

## Quick Start

1. Clone the repository:

```bash
git clone https://github.com/mikeislas06/Sequence-Companion-App-server.git
cd Sequence-Companion-App-server
```

2. Install dependencies:

```bash
npm install
# or
pnpm install
```

3. Create a `.env` file at the root of the `server` folder:

```env
PORT=3001
CLIENT_URL=http://localhost:3000
```

4. Start the development server:

```bash
npm run dev
```

The server will start on [http://localhost:3001](http://localhost:3001).  
Test it is running by visiting [http://localhost:3001/health](http://localhost:3001/health) — you should see `{ "ok": true }`.

## Environment Variables

| Variable     | Description                             | Default |
| ------------ | --------------------------------------- | ------- |
| `PORT`       | Port the server listens on              | `3001`  |
| `CLIENT_URL` | Allowed CORS origin (your frontend URL) | `*`     |

> In production, set `CLIENT_URL` to your Vercel frontend URL to restrict CORS.

## API & Socket Events

### REST

| Method | Path      | Description                                         |
| ------ | --------- | --------------------------------------------------- |
| GET    | `/health` | Returns `{ ok: true }` — used for uptime monitoring |

### Socket.IO Events (client → server)

| Event             | Payload                  | Description                                |
| ----------------- | ------------------------ | ------------------------------------------ |
| `room:create`     | `{ playerName, config }` | Create a new room                          |
| `room:join`       | `{ code, playerName }`   | Join an existing room by code              |
| `room:leave`      | —                        | Leave the current room                     |
| `game:start`      | —                        | Host starts the game and deals cards       |
| `card:play`       | `{ cardId }`             | Play a card from hand                      |
| `card:dead`       | `{ cardId }`             | Mark a card as dead and draw a replacement |
| `sequence:update` | `{ teamIndex, delta }`   | Update sequence row count for a team       |
| `game:reset`      | —                        | Reset room state for a new game            |

### Socket.IO Events (server → client)

| Event          | Description                                      |
| -------------- | ------------------------------------------------ |
| `room:updated` | Full updated room state broadcast to all players |
| `game:started` | Game has begun, includes initial hand            |
| `game:over`    | Win condition met, includes winning team         |
| `hand:updated` | Player's private hand updated after card action  |
